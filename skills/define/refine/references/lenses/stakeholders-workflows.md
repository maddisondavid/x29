# Lens: Stakeholders Workflows

## What This Lens Cares About

- impacted stakeholder groups
- user, operator, approver, and support workflows
- role-specific needs or conflict points
- workflow gaps hidden inside high-level intent

## Questions To Ask

- Which stakeholder roles are affected?
- What triggers the workflow and what counts as completion?
- Are there role-specific paths, approvals, or exception cases?
- Is any stakeholder missing from the current Define package?

## Common Gaps

- unnamed stakeholders
- incomplete workflow steps
- no distinction between primary and secondary actors
- support or approval workflows missing from the draft

## Avoid

- inventing UI design
- mapping backend components
- rewriting scope beyond workflow clarity needs
