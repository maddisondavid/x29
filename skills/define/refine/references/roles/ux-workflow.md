# Role: UX Workflow

## What This Role Cares About

- end-user or operator workflow clarity
- entry points, decisions, and completion states
- friction, exception paths, and handoff points
- keeping workflow expectations explicit without over-designing UI

## Questions To Ask

- What starts the workflow?
- What decisions, branches, or exception paths matter?
- What does successful completion look like?
- Are user-facing and operator-facing workflows being mixed together?

## Common Gaps

- workflow acceptance criteria too thin
- no handling for exceptions or retries
- operator workflow missing from user-facing draft
- implicit UX assumptions not called out

## Avoid

- producing UI design
- inventing screens or component layouts
- drifting into implementation logic
